## Getting Unstuck

If you hit a wall:

1. Re-read the relevant handbook section
2. Run the demo code, modify it, experiment
3. Review Project 5 (RPG Arena)—it demonstrates all concepts in a simpler form
4. Attend office hours (schedule in syllabus)
5. Ask classmates (explain your approach, not your code)

**Don't stay stuck for more than 30 minutes.** If you're spinning, get help.

## Grading

Your final grade is based solely on **Project 6 (IMS)**. See **[`final_project_rubric.md`](./final_project_rubric.md)** for detailed grading criteria.

- **Functionality (40%):** All features work as specified

- **Integration (20%):** Combines OOP + functions + file I/O + exceptions
- **Code Quality (10%):** Clear names, docstrings, edge case handling
